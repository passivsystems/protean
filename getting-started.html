Installing

Linux

There is a tried and tested .deb for Debian/Ubuntu flavours of Linux at : https://github.com/passivsystems/protean/releases/download/0.9.0-pre.7/protean_0.9.0_all.deb

Mac OSX

For OSX we have a brogrammer script, trust us, it is safe :-) This will install to ~/bin and in a very impolite way spew a couple of files out in there. In a terminal run bash <(curl -fksSL https://raw.githubusercontent.com/passivsystems/protean/develop/install/install.sh)

Starting

In a terminal run protean-server. This will start the sim server. You will see some output when you run this startup command, including information on where service definitions live.

There are several configuration possibilities via environment variable:

  • PROTEAN_SIM_PORT - the port for hitting sim endpoints (default 3000)
  • PROTEAN_ADMIN_PORT - the port for managing sims (default 3001)
  • PROTEAN_CODEX_DIR - the location for codex specifications (listed when you start the sim server)
  • PROTEAN_LOG_LEVEL - (default info)
  • PROTEAN_LOG_DIR - (default current working directory)
  • PROTEAN_TARGET - the location for produced API docs etc (default relative target directory)

API Documentation Generation

It is simple to generate API docs from your codex.

In a terminal run protean doc -f myservice.cod.edn.

The output in the terminal will tell you how to open your docs.

You will notice in the produced docs that values have been generated where you have specified types for parameters. This is true in the curl output also. See the sample petstore codex for examples of types.

Simulation

Protean ships with a sample 'petstore' service for you to experiment with. See sample-petstore.cod.edn in the 'codex' directory which is listed in the output when you run the startup command.

You can use a command line interface CLI to play with Protean. In a terminal run protean to get some help with things.

  • Run protean services to list all services.
  • Run protean service-usage -n petstore to get a list of curl commands you can hit the service with.
  • Add your own service to Protean by running protean add-services -f myservice.cod.edn.

Integration Testing

It is possible to integration test your simulations in a fairly simplistic manner. You do not need to write any code to do this.

In a terminal run protean test -f myservice.cod.edn.

The output in the terminal will indicate success or failure as the endpoints you have specified in your codex are traversed.

We are currently working on integration testing your actual services, not the simulations. We will update these docs when we have more news on that.
